The main server application wraps the parser for the payloads.
    It is self-threading to allow as many connections simultaneously
    as needed and can close stale connections in order to not
    waste resources. It is in charge of recieving the payloads
    and passing them to the parser directly, recieving a return from
    the parser and returning the return payload. The parser is expected
    to take care of error handling and handling invalid payloads.


The payloads that get sent from client to server and returned
    back to the client are layed out very specifically. Since we
    are using Object Streams from the java.net package, we need to
    define the payload to be as small of a class as possible to reduce
    overhead. Some of the variables are reused a couple of times and
    our general layout of the payloads is described here

The MySQL server will house the database and it's tables in order to
