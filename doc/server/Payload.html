Payload Values


CLIENT/SERVER INTERACTIONS

  • Check Deliverability

    • (client send)Send LatLng, opcode 0, value 0

    • (server do) Find Nearest Store

      • check distance between client and store

      • check weather of store

      • check open status of store

      • check drones at store

    • (server return) return storeID in value if good. negative value if bad. opcode=0

    • (client receive) store the received storeID

  • Place order

    • (client send) Send name=user’s name, value=storeID, opcode=1, weight=weight

    • (server receive) place order

      • Assign order an ID

      • Store order ID in store table (confirm it exists)

      • Wait for store to update order ID list

      • Path A: Order is Denied (Found in store with bad value)

        • Sent value for orderID

      • Path B: Order is Accepted

        • Sent value for orderID

        • Store orderID information

        • Update user information with new order ID

    • (server return) return newly created orderID=value, opcode=1 (negative orderID if failed)

    • (client receive) parse orderID and add to order list

  • Get Order Status

    • (client send) Send value=orderID, opcode=2

    • (server receive) retrieve order status

      • Path A: Order is Complete (Find order ID and check Completion Date )

        • Set values

      • Path B: Order is Bad (No order ID is found)

        • Set Values

      • Path C: Order is not Complete (same as Path A but date is 0)

        • Get store ID from order table with order ID

        • Check status of drone in store object

        • Set values

    • (server return) value=good or bad, opcode=2, LatLng of drone, weight=ETA

    • (client receive) display status of order on map and overlay
